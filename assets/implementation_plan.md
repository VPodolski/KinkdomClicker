# План реализации загрузочного экрана

В этом плане описано, как мы реализуем экран загрузки с прогресс-баром и заглушкой для стартового баннера при запуске игры.

## Требуется ваше решение

- **Заглушка для баннера**: Вы хотите, чтобы я сгенерировал красивую картинку с помощью нейросети (например, пиксельный арт замка) для этого баннера, или пока просто сделать пустой серый прямоугольник с текстом "Место для баннера", чтобы вы позже заменили его на свой арт?

## Предлагаемые изменения

### 1. Новая сцена: `UI/loading.tscn`
Мы создадим новую сцену, которая станет стартовой точкой игры. В ней будут:
- `ColorRect` на весь экран для темного фона.
- `TextureRect` (или `ColorRect` в качестве заглушки) в центре экрана под стартовый баннер.
- `ProgressBar` (полоса загрузки) в нижней части экрана для отображения прогресса.

### 2. Новый скрипт: `Scripts/Loading.gd`
Мы напишем скрипт для управления экраном загрузки.
- Он будет использовать встроенную в Godot функцию `ResourceLoader.load_threaded_request` для асинхронной загрузки основной сцены (`res://main.tscn`) в фоновом режиме.
- Скрипт будет каждый кадр обновлять значение `ProgressBar.value` с помощью `ResourceLoader.load_threaded_get_status`.
- Мы добавим небольшую искусственную задержку (около 1.5 - 2 секунд), чтобы игрок успел увидеть загрузочный экран и баннер, так как игра может загружаться слишком быстро.
- Как только загрузка завершится, скрипт автоматически переключит сцену с помощью `get_tree().change_scene_to_packed()`.

### 3. Конфигурация проекта: `project.godot`
#### [ИЗМЕНЕНИЕ] `project.godot`
Мы изменим параметр `run/main_scene` с `main.tscn` на `UI/loading.tscn`. Это гарантирует, что игра всегда будет начинаться с экрана загрузки.

## План проверки

### Ручная проверка
- Запустить игру.
- Убедиться, что сначала появляется экран загрузки с баннером.
- Проверить, что полоса загрузки плавно заполняется до 100%.
- Убедиться, что происходит бесшовный и корректный переход к основному интерфейсу игры.
